Backend Requirements

Database Tables:

menu_items:

id, name, description, price (USD), category, image_url, available (boolean)

reservations:

id, name, email, phone, date, time, guests, notes, status (pending/confirmed/cancelled), created_at

comments (testimonials):

id, customer_name, rating, comment, approved (boolean), created_at

orders:

id, customer_name, email, phone, delivery_address, city, postal_code, delivery_instructions, subtotal, delivery_fee, total, status, created_at

order_items:

id, order_id (foreign key), item_name, quantity, price

history (timeline):

id, year, title, description

API Endpoints:

GET /api/menu - Get all menu items (filtered by available=true)GET /api/menu/:category - Get items by category

POST /api/reservations - Create reservationGET /api/reservations - Get all reservations (admin)

GET /api/testimonials - Get approved testimonialsPOST /api/testimonials - Submit new testimonial (starts as approved=false)

POST /api/orders - Create orderGET /api/orders/:id - Get order details

GET /api/history - Get timeline items
